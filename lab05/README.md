## Описание лабораторной работы
Лабораторная работа посвящена объектно-ориентированному программированию в PHP.
В рамках работы реализовано приложение для управления банковскими транзакциями с использованием классов, инкапсуляции, интерфейсов и разделения ответственности между компонентами.

Цель работы: освоить создание собственных классов в PHP, работу с объектами, репозиториями, интерфейсами и выводом данных в HTML-таблицу.

## Краткая документация к проекту
Структура проекта:
- `index.php` — основная реализация лабораторной работы.
- `task.md` — формулировка задания.
- `report-requirements.md` — требования к отчету.

Основные сущности и классы:
- `Transaction` — класс одной банковской транзакции. Содержит приватные свойства `id`, `date`, `amount`, `description`, `merchant`, `category`, getter-методы и `getDaysSinceTransaction(): int`.
- `TransactionStorageInterface` — интерфейс для абстракции хранилища транзакций.
- `TransactionRepository` — класс, который хранит массив объектов `Transaction`, добавляет, удаляет, возвращает и ищет транзакции по `id`.
- `TransactionManager` — класс бизнес-логики. Считает суммы, фильтрует транзакции по дате, считает количество транзакций по получателю и сортирует данные.
- `TransactionTableRenderer` — отдельный `final` класс для генерации HTML-таблицы транзакций.

Основные методы:
- `addTransaction(Transaction $transaction): void` — добавляет объект транзакции в репозиторий.
- `removeTransactionById(int $id): void` — удаляет транзакцию по идентификатору.
- `getAllTransactions(): array` — возвращает список всех транзакций.
- `findById(int $id): ?Transaction` — ищет транзакцию по `id`.
- `calculateTotalAmount(): float` — считает общую сумму всех транзакций.
- `calculateTotalAmountByDateRange(string $startDate, string $endDate): float` — считает сумму транзакций за диапазон дат.
- `countTransactionsByMerchant(string $merchant): int` — считает количество транзакций по получателю.
- `sortTransactionsByDate(): array` — сортирует транзакции по дате.
- `sortTransactionsByAmountDesc(): array` — сортирует транзакции по сумме по убыванию.
- `render(array $transactions): string` — формирует HTML-таблицу для списка транзакций.

## Примеры использования проекта (фрагменты кода)
Создание объекта транзакции:
```php
$transaction = new Transaction(
    1,
    '2025-01-10',
    125.40,
    'Payment for groceries',
    'SuperMart',
    'Groceries'
);
```

Добавление транзакции в репозиторий:
```php
$repository = new TransactionRepository();
$repository->addTransaction($transaction);
```

Создание менеджера и подсчет общей суммы:
```php
$manager = new TransactionManager($repository);
$totalAmount = $manager->calculateTotalAmount();
```

Сортировка транзакций по сумме:
```php
$sortedByAmountDesc = $manager->sortTransactionsByAmountDesc();
```

Вывод HTML-таблицы:
```php
$renderer = new TransactionTableRenderer();
echo $renderer->render($repository->getAllTransactions());
```

Поиск транзакции по ID:
```php
$foundTransaction = $repository->findById(4);
```

Удаление транзакции:
```php
$repository->removeTransactionById(3);
```

## Ответы на контрольные вопросы
1. Зачем нужна строгая типизация в PHP и как она помогает при разработке?
Строгая типизация включает более строгую проверку передаваемых и возвращаемых значений. Она помогает раньше обнаруживать ошибки, делает код понятнее и уменьшает вероятность неявных преобразований типов.

2. Что такое класс в объектно-ориентированном программировании и какие основные компоненты класса вы знаете?
Класс — это шаблон для создания объектов. Основные компоненты класса: свойства, методы, конструктор, модификаторы доступа (`public`, `private`, `protected`) и, при необходимости, константы.

3. Объясните, что такое полиморфизм и как он может быть реализован в PHP.
Полиморфизм — это возможность работать с разными объектами через общий интерфейс или общий базовый тип. В PHP он реализуется через интерфейсы, наследование и переопределение методов. В данной работе `TransactionManager` работает с `TransactionStorageInterface`, а не с конкретным классом хранилища.

4. Что такое интерфейс в PHP и как он отличается от абстрактного класса?
Интерфейс задает контракт: какие методы класс обязан реализовать. Он не хранит состояние объекта и не содержит обычной реализации свойств. Абстрактный класс может содержать как абстрактные, так и обычные методы, а также свойства. Интерфейс нужен для определения общего поведения, а абстрактный класс — для частично готовой базовой реализации.

5. Какие преимущества дает использование интерфейсов при проектировании архитектуры приложения? Объясните на примере данной лабораторной работы.
Интерфейсы уменьшают связанность кода и делают архитектуру гибче. В этой лабораторной `TransactionManager` зависит от `TransactionStorageInterface`, поэтому при необходимости можно заменить `TransactionRepository` на другое хранилище без изменения бизнес-логики менеджера.

## Дополнительные важные аспекты
- В проекте используется `declare(strict_types=1);`.
- Все свойства класса `Transaction` объявлены как приватные и доступны только через getter-методы.
- Для вычисления количества дней с момента транзакции используется `DateTime`.
- Для большей гибкости добавлен интерфейс `TransactionStorageInterface`.
- Для соответствия требованию таблицы в модель транзакции добавлено поле `category`.
- Проект носит учебный характер и не использует базу данных или внешние библиотеки.
