# Лабораторная работа №7. Шаблонизация

## Цель работы

Освоить принципы шаблонизации в PHP, как с использованием нативных PHP-шаблонов, так и с применением готового шаблонизатора. Улучшить структуру проекта, разделив логику обработки данных и представление.

## Условие

Продолжите разработку проекта из _лабораторной работы №6_, разделив его на слои: логику и представление. Работа выполняется в два этапа - сначала вы реализуете собственный механизм шаблонизации, затем подключаете Twig и реализуете ту же функциональность через него.

### Шаг 1. Рефакторинг. Нативные PHP-шаблоны

Перепишите существующий проект, разделив код на два уровня:

1. _Логика_ - PHP-файлы, которые обрабатывают запросы, читают/записывают данные и подготавливают переменные.
2. _Представление_ - отдельные `.php`-файлы-шаблоны, которые только отображают данные.

Один из примеров структуры проекта может выглядеть так:

```
project/
├── templates/
│   ├── layout.php       # Общий макет (header, footer)
│   ├── form.php         # Шаблон формы добавления записи
│   └── list.php         # Шаблон таблицы со списком записей
├── src/
│   ├── functions.php    # Функции для работы с данными
│   └── handler.php      # Обработчики запросов (например, для добавления новой записи)
├── index.php            # Точка входа, логика маршрутизации
└── data.json            # Файл с данными
```

> [!NOTE]
>
> Вы можете продолжать использовать ту же структуру проекта, что и в лабораторной работе №6, но обязательно выделите шаблоны в отдельную папку и убедитесь, что они не содержат логики обработки данных.

### Шаг 2. Использование готовых шаблонизаторов

2.1. Установите и настройте один из готовых шаблонизаторов, например, [Twig](https://twig.symfony.com/), с помощью Composer.
2.2. Перепишите те же шаблоны, используя синтаксис выбранного шаблонизатора.
2.3. Обязательно используйте возможности шаблонизатора для улучшения структуры шаблонов (например, наследование шаблонов, блоки и т.д.).

В проекте должно быть показано использование как нативных PHP-шаблонов, так и готового шаблонизатора. Оба варианта должны присутствовать в проекте, чтобы можно было сравнить их между собой.

### Шаг 3. Дополнительные функции

> [!TIP]
>
> При выполнении всех предыдущих шагов максимальная оценка составляет 9 баллов. Для получения 10 баллов необходимо реализовать одно из следующих дополнительных заданий.

Создайте и зарегистрируйте собственный фильтр Twig, который решает практическую задачу в вашем проекте. Например:

- `{{ prep_time | format_duration }}` → `"1 ч 30 мин"` для рецептов;
- `{{ amount | currency('USD') }}` → `"$1 234.56"` для трекера расходов;
- `{{ mood | mood_icon }}` → `"☺️"` (_эмодзи_) для дневника настроения.
